Term Reflection

Retrobowl Sandlot

This term focuses on a concise, sandlot-inspired football slice that highlights decision making, crisp ball handling, and an inviting presentation. Rather than chase full-season simulation, I’m polishing one expressive possession that showcases input design, feedback loops, and my iteration cadence.

Key Progress

  • Lightweight browser build with a cohesive retro-stadium art direction.
  • Readable route overlays, player markers, and HUD callouts for situational clarity.
  • Automated GitHub Pages deployment for effortless stakeholder demos.
  • Up-to-date documentation, screenshots, and reflection captured in this page.

What Is Going Well

The focused scope keeps iteration nimble, and the current visual language resonates during critiques. Controls remain responsive on keyboard/mouse, and the deploy pipeline means I can circulate builds with a single link instead of screen-sharing.

Challenges & Lessons

  • Constant scope triage—new feature ideas must not dilute the core slice.
  • SVG route animation required deeper planning, reinforcing the value of sketches.
  • Workflow automation took research but now accelerates every review cycle.

Plan for the Remainder of the Term

  1. Polish passing feel with richer feedback (meter, catch confirmation, ball flight).
  2. Prototype at least one new defensive behavior for more dynamic reads.
  3. Add a lightweight tutorial sequence before the first possession.
  4. Playtest with classmates and fold their friction notes into the loop.

Game Overview

Retrobowl Sandlot is a browser-based vignette blending arcade pacing with a light season loop. Mouse-aimed throws, minimal HUD chrome, and lively receiver routes make each possession feel immediate, while broadcast-inspired overlays maintain a professional sheen.

Feature Highlights

  • Mouse-aimed throwing with a hold-to-charge arc that clears defenders.
  • Receiver route overlays drawn in SVG for instant pre-snap clarity.
  • Animated trenches suggesting pocket dynamics even in this slice.
  • Assignment-based defense so DBs trail their receiver, not just the ball.
  • Persistent LOS + chains to keep down-and-distance visible.
  • Eight-week season loop culminating in rings for every title run.

Controls at a Glance

Action Input
Move QB / ball carrier Arrow Keys
Sprint Shift
Throw Click + hold to charge, release toward the cursor
Reset season R

Tip: Control transfers to the receiver automatically after a catch.

Season & Progression

  • Schedule generation: curated opponents scale with season and week difficulty.
  • Playoffs: winning records unlock semifinal and championship rounds with trophy rings.
  • Dynamic branding: HUD labels and away end zone art update per opponent.

Project Structure

style.css    # Stadium aesthetic, player markers, responsive layout

README.md    # Project documentation
README.html  # Reflection + documentation page

Development Workflow

  1. Clone or download the repository.
  2. Open index.html in a modern browser—no build step required.
  3. Optionally use python -m http.server or npx serve for live reload.

Deployment Status

  • GitHub Pages serves the project directly from main.
  • Workflows publish README.html whenever documentation changes.
  • Stakeholders access the latest build via one consistent URL.
